# AI Rules (Strict)
1. **Behavior**: No apologies, summaries, feedback on understanding, or unnecessary confirmations. Be concise.
2. **Logic**: Verify info; no assumptions, speculations, or inventions. Handle edge cases/errors robustly.
3. **Coding**: Preserve existing structure; use explicit names and consistent style. Prioritize performance, security, and modularity. Use assertions. Include unit tests.
4. **Editing**: File-by-file, single-chunk edits only. No whitespace-only or unnecessary updates. Don't discuss current implementation unless asked.
5. **Context**: Link real files. Check context-generated files for implementation details.
6. **Shell**: All terminal commands must use **fish-shell** syntax.
